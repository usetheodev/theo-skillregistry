## Release notes (summary)

### Added
- M1 — Skill model + rigorous validation: Theokit-compatible SKILL.md frontmatter parser
  (`yaml`/eemeli), zip-safety validation (`yauzl`; traversal/symlink/zip-bomb/depth/dup/SKILL.md-root)
  + secret scan (`secretlint`), immutable revisions (bytea + sha256 + jsonb), full CRUD
  (paginated list, updateMask, delete + configurable id reservation, revisions endpoints).

### Changed
- Frontmatter YAML parser switched gray-matter → `yaml` (eemeli) to avoid the js-yaml 3.x CVE.

### Fixed
- A deleted skill_id can be recreated after the reservation window expires (BLOCKER from /review).
- Index on `skill_revisions(skill_id, create_time desc)`.

### Security
- Inbound body-size limit (413) on POST/PATCH — memory-DoS guard (`THEOSKILL_MAX_BODY_BYTES`).
