## Release notes

### Added
- M9: rastreabilidade ponta-a-ponta — `trace_id` W3C-compatível originado na fronteira HTTP, propagado
  por operação → job → webhook, logado em cada salto e persistido na delivery row (sobrevive ao
  re-enqueue do reconciler). Seam mínimo (`node:crypto`, sem SDK OpenTelemetry — o M8 adota).
- M9: CLI `theoskill init` (.theoskillrc 0600, auth nunca impresso) + comandos de leitura
  `get`/`list`/`status`/`revisions`; flags caem para a config.
- M9: taxonomia de test markers (`[slow]`/`[live]`/`[integration]`) + `test:fast`.
- M9: seleção de embedder vira registry ordenado OCP (3º provider YAGNI-deferido, ADR-3).

### Changed
- M9: backoff de webhook agora é política explícita (`WEBHOOK_DELIVERY_BACKOFF`) com `computeBackoff` puro.
- Template de instalação (`.claude/settings.json`): `bypassPermissions` + ferramentas de pacote em `allow`.

### Fixed
- `/discover-plan-confidence`: threshold parser lê o formato `KEY = VALUE` documentado (ADR 0001).

### Security
- M9: o logger redige valores de chaves sensíveis (authorization/password/token/secret + sufixos) antes de emitir.
